DevOps is a way of working where the Development and Operations teams collaborate closely throughout the software development process. Instead of working separately, both teams work together to build, test, deploy, and maintain applications.
In traditional development, developers write the code and then pass it to the operations team for deployment. This often leads to communication gaps, delays, and deployment issues. As a result, software releases can take a long time and fixing problems becomes difficult.
DevOps solves these issues by encouraging teamwork and using automation tools. It helps organizations release software faster, reduce errors, improve collaboration, and deliver better products to users.

Continuous Integration (CI)
Continuous Integration is the practice of automatically building and testing code whenever developers make changes to the project. The main goal is to find and fix issues early.
Example:
Suppose a developer adds a new feature to a login page and pushes the code to GitHub. The CI pipeline automatically builds the application and runs test cases to check whether everything is working correctly. If a test fails, the developer is notified immediately.
Continuous Delivery/Deployment (CD)
Continuous Delivery or Continuous Deployment focuses on releasing the tested code to different environments such as staging or production.
Example:
After the login feature passes all tests in the CI stage, the CD pipeline automatically deploys the updated application to a server where users can access the new feature.
